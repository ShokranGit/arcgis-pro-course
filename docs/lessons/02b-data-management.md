# Lesson 02b · Data Acquisition and Management

## Learning goals

- Organize project folders and files using a consistent, predictable structure.
- Identify reliable sources for GIS data, including open data portals and institutional repositories.
- Acquire a dataset relevant to a project area and evaluate its quality and metadata.
- Add acquired data to a map in ArcGIS Pro and confirm it displays correctly.
## Key concepts

Good file management is invisible when it works and painful when it doesn't. Keep raw downloaded data separate from processed outputs, use short and consistent folder and file names, and avoid moving a geodatabase or shapefile's component files independently — GIS data formats are often made of several linked files that all need to travel together.

Not all data is equally trustworthy. When acquiring a dataset, check its source, its last update date, its coordinate system, and its documentation or metadata before relying on it for analysis.

## Workflow

1. Create a clean folder structure for your project (for example: data, maps, exports).
2. Explore at least two public GIS data portals or repositories.
3. Download a dataset relevant to your project area, noting its source and any license terms.
4. Add the downloaded data to a new or existing map in ArcGIS Pro.
5. Confirm the data draws correctly and review its attribute table for obvious problems.
## Practice task

Download one vector or raster dataset from a public data source and add it to a map project. Note the source, download date, and coordinate system in a short caption.

## Deliverable

Submit a screenshot of your organized project folder and a screenshot of the map with your new data added.
